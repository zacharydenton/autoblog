WordPress 3.1 Release Candidate 3 is now available. After careful evaluation of the 3.1 features of RC2, we realized the need to make some adjustments. There are some important differences from previous versions of 3.1, so you can read the changes if you have developed for a beta or RC version.

The biggest change is the removal of AJAX list tables, which was an attempt to move all of our list-style screens to full AJAX pagination, search and sort column and to consolidate the list-style displays to a single API that plugins could use. Unfortunately, with more testing came realizations that there were too many major bugs and usability problems with the functionality was implemented, so we have spent the last week to roll back the main elements of the function.

  • For users: AJAX has been totally disabled for the list tables. We hope to get it back again, in a form that is properly and fully applied in future versions. Column sorting remain, but everything else is back to its 3.0 state.
  • For developers: The whole list table API is now marked private. If you try to take advantage of new components of the API, you're pretty much guaranteeing that your plugins will break in a future version, so you do not do it. We hope to be able to do all the fun new goodies for general use in a future version.

This is the only way we could prevent regressions in functionality and usability of WordPress 3.0 to 3.1. That's right, users and plugin writers can still do everything you used to do (and then some).

Because of the code churn between RC2 and RC3, release candidate, this must be a lot of testing. Each list screen-tested. In particular, the comment moderation screen testing, especially with keyboard shortcuts (if you did not know about them, now's your chance to try them).

Other patches in the RC3 includes:

  • Correctly display the author shortcut menu Quick Edit
  • Various important fixes to the many variables taxonomy question
  • Fixes to the theme of the removal process
  • Fixes to the pages that are used for posts
  • IIS and Multisite: Do not restore the web.config to save permalink
  • Properly validate mail format and their rewrite rules

I assemble a group of friends in Washington, DC, this weekend to test WordPress 3.1 and provide feedback. We'd love to see this idea catching among friends at coffee shops around the world. (We will be blogging our results, and we are thinking of using hashtag # wptest on Twitter.) If you test the release candidate and believe that you have found a bug, there is no way to tell us:

  • Post on the Alpha / Beta area of support forums
  • Report it to the wp-testers mailing list
  • Participate in the development IRC channel and talk live on irc.freenode.net # wordpress-dev
  • A bug ticket on the WordPress Trac

In order to test WordPress 3.1, try the WordPress Beta Tester plugin (you will want to "bleeding edge Night Lies"). Or you can download the release candidate here (zip).

We will study this issue carefully to see where we can improve our internal processes in the future. With this, our requisite haiku, written by Jane:
